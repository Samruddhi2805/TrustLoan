# TrustLoan Lite - User Feedback Form Template

## Google Form Structure

### Form Title: "TrustLoan Lite DApp - User Feedback"

### Form Description:
"Thank you for testing TrustLoan Lite! Your feedback helps us improve the DApp experience. Please answer the following questions about your experience with our loan eligibility checker."

---

## Questions to Include:

### 1. Wallet Address (Required)
- **Type**: Short text
- **Description**: "Enter your wallet address (e.g., 0x1234...abcd)"
- **Validation**: Must match Ethereum address format

### 2. Email Address (Required)
- **Type**: Short text
- **Description**: "For follow-up questions and updates"

### 3. Name (Optional)
- **Type**: Short text
- **Description**: "Your name (optional)"

### 4. Overall Experience Rating (Required)
- **Type**: Linear scale (1-5)
- **Labels**: 1 = Very Poor, 5 = Excellent
- **Description**: "Rate your overall experience with the DApp"

### 5. Ease of Use (Required)
- **Type**: Linear scale (1-5)
- **Labels**: 1 = Very Difficult, 5 = Very Easy
- **Description**: "How easy was it to connect your wallet and check eligibility?"

### 6. UI/UX Rating (Required)
- **Type**: Linear scale (1-5)
- **Labels**: 1 = Dislike, 5 = Love
- **Description**: "Rate the visual design and user interface"

### 7. Transaction Speed (Required)
- **Type**: Linear scale (1-5)
- **Labels**: 1 = Very Slow, 5 = Very Fast
- **Description**: "How was the transaction processing speed?"

### 8. Features Used (Required)
- **Type**: Checkboxes
- **Options**:
  - [ ] Wallet connection
  - [ ] Income/expense form
  - [ ] Eligibility check
  - [ ] Result viewing
  - [ ] Transaction hash verification

### 9. What did you like most? (Optional)
- **Type**: Paragraph
- **Description**: "What features or aspects of the DApp did you enjoy the most?"

### 10. What needs improvement? (Optional)
- **Type**: Paragraph
- **Description**: "What features or aspects need improvement?"

### 11. Technical Issues (Required)
- **Type**: Multiple choice
- **Options**:
  - [ ] No issues
  - [ ] Wallet connection problems
  - [ ] Transaction failures
  - [ ] UI/UX issues
  - [ ] Slow loading
  - [ ] Other (please specify)

### 12. Browser Used (Required)
- **Type**: Multiple choice
- **Options**:
  - [ ] Chrome
  - [ ] Firefox
  - [ ] Safari
  - [ ] Edge
  - [ ] Other

### 13. Mobile/Desktop (Required)
- **Type**: Multiple choice
- **Options**:
  - [ ] Desktop
  - [ ] Mobile
  - [ ] Tablet

### 14. Would you use this again? (Required)
- **Type**: Multiple choice
- **Options**:
  - [ ] Definitely yes
  - [ ] Probably yes
  - [ ] Maybe
  - [ ] Probably not
  - [ ] Definitely not

### 15. Additional Comments (Optional)
- **Type**: Paragraph
- **Description**: "Any other feedback or suggestions?"

## Data Analysis Plan:

### Metrics to Track:
- **Average Rating**: Overall experience (target: 4.0+/5.0)
- **Completion Rate**: % of users who complete full form
- **Feature Usage**: Most/least used features
- **Common Issues**: Frequently reported problems
- **Device Distribution**: Mobile vs Desktop usage

### Improvement Areas:
- Ratings below 3.5 indicate need for improvement
- Comments about wallet connection need priority
- UI/UX feedback for design iterations
- Transaction speed issues for optimization

---

## Follow-up Actions:
1. **Weekly Review**: Analyze new responses
2. **Priority Matrix**: High-impact, low-effort fixes first
3. **User Communication**: Email updates on improvements
4. **Iteration Planning**: Use feedback for next development cycle
